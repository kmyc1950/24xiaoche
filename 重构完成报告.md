## 🔑 核心技术要点

### 1. 硬件参数配置

```c
// 编码器参数
#define ENCODER_LINE            13          // 编码器线数
#define MOTOR_REDUCTION_RATIO   28          // 减速比 1:28
#define ENCODER_PPR             364         // 每转脉冲数

// 车轮参数
#define WHEEL_DIAMETER_MM       65.0f       // 车轮直径
#define WHEEL_CIRCUMFERENCE_MM  204.2f      // 车轮周长

// 采样周期
#define PID_SAMPLE_TIME_MS      50.0f       // 50ms周期
#define PID_SAMPLE_FREQ_HZ      20.0f       // 20Hz频率

// PWM参数
#define PWM_MAX_DUTY            4000        // 最大占空比
```

### 2. 测速换算公式

**公式推导**：
```
速度(mm/s) = 脉冲增量 × 车轮周长 / 每转脉冲数 / 采样时间
           = pulse × 204.2 / 364 / 0.05
           = pulse × 11.22
```

**代码实现**（motor.c: 161-171行）：
```c
Motor_Left.current_speed = (float)encoder_counter_left 
                            * WHEEL_CIRCUMFERENCE_MM 
                            / ENCODER_PPR 
                            / PID_SAMPLE_TIME_S;
```

### 3. PI控制算法

**位置式PI公式**：
```
PWM(k) = Kp × error(k) + Ki × Σerror(k)
```

**积分限幅**（motor.c: 203-209行）：
```c
motor->integral += motor->error;

// ★★★ 积分限幅（防止积分饱和）★★★
if (motor->integral > motor->integral_max) {
    motor->integral = motor->integral_max;    // 上限 2000
} else if (motor->integral < motor->integral_min) {
    motor->integral = motor->integral_min;    // 下限 -2000
}
```

**PWM输出限幅**（motor.c: 217-223行）：
```c
// ★★★ PWM输出限幅（防止超出定时器重载值）★★★
if (pwm_value > (int32_t)PWM_MAX_DUTY) {
    pwm_value = (int32_t)PWM_MAX_DUTY;        // 限制为 4000
} else if (pwm_value < -(int32_t)PWM_MAX_DUTY) {
    pwm_value = -(int32_t)PWM_MAX_DUTY;       // 限制为 -4000
}
```

### 4. 定时器中断闭环

**PID定时器中断**（50ms周期，motor.c: 276行）：
```c
void MOTOR_PID_INST_IRQHandler(void)
{
    // 左轮闭环
    calculate_speed(1);      // 1. 读取编码器脉冲，计算速度
    motor_pi_loop(1);        // 2. 执行PI算法，更新PWM

    // 右轮闭环
    calculate_speed(2);
    motor_pi_loop(2);
}
```

**完整闭环流程**：
```
[定时器中断 50ms]
       ↓
[读取编码器脉冲] → [计算物理速度] → [清零计数器]
       ↓
[计算误差: error = target - current]
       ↓
[累加积分: integral += error]
       ↓
[★积分限幅★]
       ↓
[PI计算: pwm = Kp×error + Ki×integral]
       ↓
[★PWM限幅★]
       ↓
[根据pwm正负设置方向和占空比]
       ↓
[更新电机PWM] → [等待下次中断]
```

### 5. GPIO中断处理

**编码器 + 按键共享中断**（motor.c: 302行）：
```c
void GROUP1_IRQHandler(void)
{
    switch (DL_GPIO_getPendingInterrupt(DC_MOTOR_LEFT_AA_PORT))
    {
    case DC_MOTOR_LEFT_AA_IIDX:
        // 左轮编码器A相中断
        {
            uint32_t pin_b = DL_GPIO_readPins(DC_MOTOR_LEFT_AB_PORT, DC_MOTOR_LEFT_AB_PIN);
            if (pin_b == 0) {
                encoder_counter_left++;  // 正转
            } else {
                encoder_counter_left--;  // 反转
            }
        }
        break;

    case KEY_KEY1_IIDX:
        // 按键中断（保留原有逻辑）
        {
            extern int status;
            status = (status + 1) % 3;
        }
        break;
    }
}
```

## 🐛 已修复的问题

### 问题1：中断处理函数重复定义
**现象**：编译错误 `redefinition of 'GROUP1_IRQHandler'`  
**原因**：key.c 和 motor.c 中都定义了该函数  
**解决**：删除 key.c 中的定义，在 motor.c 中统一处理

### 问题2：中断宏定义缺失
**现象**：编译错误 `'KEY_INT_IRQN' undeclared`  
**原因**：key.h 中未定义该宏  
**解决**：添加 `#define KEY_INT_IRQN GPIO_MULTIPLE_GPIOA_INT_IRQN`

### 问题3：引脚宏定义错误
**现象**：编译错误 `'DC_MOTOR_STBY_PIN' undeclared`  
**原因**：使用了不带后缀的旧宏名  
**解决**：全部改为带后缀的正规宏（如 `DC_MOTOR_LEFT_STBY_PIN`）

### 问题4：重复使能中断
**现象**：逻辑冗余  
**原因**：main.c 中分别使能了 KEY_INT_IRQN 和 DC_MOTOR_INT_IRQN  
**解决**：只使能一次 `DC_MOTOR_INT_IRQN`（它们是同一个中断）

## 📊 PI参数配置

### 默认参数
```c
Motor_Left = {
    .kp = 8.0f,              // 比例系数
    .ki = 0.5f,              // 积分系数
    .integral_max = 2000.0f, // 积分上限
    .integral_min = -2000.0f // 积分下限
};
```

### 调优建议

**响应慢、无法达到目标速度**：
- 增大 Kp（如 10.0、12.0）
- 增大 Ki（如 0.8、1.0）

**震荡明显、不稳定**：
- 减小 Kp（如 6.0、5.0）
- 减小 Ki（如 0.3、0.2）

**稳态误差大**：
- 增大 Ki（如 1.0、1.5）
- 增大 integral_max（如 2500、3000）

**启动时PWM饱和**：
- 减小 integral_max（如 1500、1000）

2. 在 motor.c 中添加 GPIOB 中断处理函数

**临时方案**：
- 当前只有左轮有编码器闭环
- 右轮可以开环控制，或手动调整 Motor_Right.kp 使其输出与左轮相近

### 2. 编码器计数方式为单边沿
**当前**：只计 A 相上升沿（364 脉冲/转）  
**升级**：可改为四倍频（1456 脉冲/转），精度提高 4 倍

### 3. 无速度滤波
**当前**：直接使用原始速度值  
**影响**：速度值可能有抖动  
**优化**：可添加滑动平均滤波或低通滤波
