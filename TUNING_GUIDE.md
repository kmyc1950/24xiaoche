## motor.c 参数详解

### 📁 文件位置
`user_driver/motor.c`

### 参数1：电机PI控制参数（速度闭环）

**位置：** motor.c 第25-42行

```c
Motor_PI_TypeDef Motor_Left = {
    .target_speed = 0.0f,
    .current_speed = 0.0f,
    .kp = 8.0f,              // ← 左轮比例系数
    .ki = 0.5f,              // ← 左轮积分系数
    .integral_max = 2000.0f,
    .integral_min = -2000.0f,
    .pwm_output = 0
};

Motor_PI_TypeDef Motor_Right = {
    .target_speed = 0.0f,
    .current_speed = 0.0f,
    .kp = 8.0f,              // ← 右轮比例系数
    .ki = 0.5f,              // ← 右轮积分系数
    .integral_max = 2000.0f,
    .integral_min = -2000.0f,
    .pwm_output = 0
};
```

#### 参数说明

**Kp（比例系数）**
- **作用：** 控制速度响应快慢
- **默认值：** 8.0f
- **调整范围：** 1.0 ~ 20.0
- **现象与调整：**
  - Kp 过大：速度超调，震荡
  - Kp 过小：响应慢，达不到目标速度
  - 建议：从 8.0 开始，根据实际效果微调

**Ki（积分系数）**
- **作用：** 消除稳态误差
- **默认值：** 0.5f
- **调整范围：** 0.0 ~ 2.0
- **现象与调整：**
  - Ki 过大：速度超调，积分饱和
  - Ki 过小：有稳态误差，达不到精确速度
  - 建议：从 0.5 开始，如果稳态误差大可适当增加到 0.8

**integral_max / integral_min（积分限幅）**
- **作用：** 防止积分饱和
- **默认值：** ±2000.0f
- **调整范围：** ±1000.0 ~ ±5000.0
- **建议：** 一般不需要调整

### 何时调整电机PI参数？

| 问题现象 | 原因 | 调整方案 |
|---------|------|---------|
| 设定300mm/s，实际只有250mm/s | Kp太小 | 增大Kp（8→10→12） |
| 速度达到后一直震荡 | Kp太大 | 减小Kp（8→6→5） |
| 有稳态误差，差5-10mm/s | Ki太小 | 增大Ki（0.5→0.8→1.0） |
| 启动时速度超调很多 | Ki太大 | 减小Ki（0.5→0.3→0.2） |
| 左右轮速度不对称 | 左右Kp/Ki不一致 | 分别调整左右轮参数 |

## huidu.c 参数详解

### 📁 文件位置
`user_driver/huidu.c`

### 参数1：巡线PD控制参数（方向控制）

**位置：** huidu.c 第11-18行

```c
// PD控制参数（可根据实际调试调整）
#define LINE_FOLLOW_KP          50.0f   // ← 比例系数
#define LINE_FOLLOW_KD          10.0f   // ← 微分系数
#define LINE_FOLLOW_BASE_SPEED  300.0f  // ← 基础速度（mm/s）

// 速度限幅（防止PI控制器输出超限）
#define SPEED_MAX               500.0f  // ← 最大速度（mm/s）
#define SPEED_MIN               0.0f    // ← 最小速度（mm/s）
```

#### 参数说明

**LINE_FOLLOW_KP（巡线比例系数）**
- **作用：** 控制转向力度
- **默认值：** 50.0f
- **调整范围：** 10.0 ~ 100.0
- **现象与调整：**
  - Kp 过大：左右摆动严重，S型行走
  - Kp 过小：转向不及时，冲出弯道
  - 建议：直线为主的赛道用较小值（30-40），弯道多的赛道用较大值（50-70）

**LINE_FOLLOW_KD（巡线微分系数）**
- **作用：** 抑制震荡，提高稳定性
- **默认值：** 10.0f
- **调整范围：** 0.0 ~ 30.0
- **现象与调整：**
  - Kd 过大：反应迟钝，转向慢
  - Kd 过小：震荡，不稳定
  - 建议：从 10.0 开始，震荡严重可增加到 15-20

**LINE_FOLLOW_BASE_SPEED（巡线基础速度）**
- **作用：** 控制小车前进速度
- **默认值：** 300.0f mm/s
- **调整范围：** 100.0 ~ 500.0 mm/s
- **现象与调整：**
  - 速度过快：冲出弯道，控制不住
  - 速度过慢：比赛时间长
  - 建议：调试阶段用 200-250，熟练后逐步提高到 350-400

**SPEED_MAX / SPEED_MIN（速度限幅）**
- **作用：** 防止差速控制时速度超限
- **默认值：** 0 ~ 500 mm/s
- **建议：** 根据电机最大速度设置，一般不需要调整

### 何时调整巡线PD参数？

| 问题现象 | 原因 | 调整方案 |
|---------|------|---------|
| 直线上左右摆动（S型） | Kp太大 | 减小Kp（50→40→30） |
| 弯道转不过来，冲出去 | Kp太小或速度太快 | 增大Kp（50→60→70）或减小BASE_SPEED |
| 转向时震荡 | Kd太小 | 增大Kd（10→15→20） |
| 转向反应慢 | Kd太大 | 减小Kd（10→5→3） |
| 直线稳但弯道差 | Kp太小 | 增大Kp |
| 弯道好但直线摆 | Kp太大 | 减小Kp |

### 参数2：穷举法误差值

**位置：** huidu.c 第76-208行（Huidu_Get_Error 函数）

```c
switch (sensor_data)
{
    case 0b00011000:  // L1 + R1（中间两个）
        error = 0.0f;  // ← 可以微调这些误差值
        break;
    
    case 0b00001100:  // L1 + L2
        error = -1.0f;  // ← 如果左转不够，改为 -1.2f
        break;
    
    case 0b00110000:  // R1 + R2
        error = 1.0f;   // ← 如果右转不够，改为 1.2f
        break;
    
    // ... 其他case
}
```

#### 何时调整误差值？

| 问题现象 | 原因 | 调整方案 |
|---------|------|---------|
| 某些状态转向力度不够 | 误差值太小 | 增大对应case的误差绝对值 |
| 某些状态转向过度 | 误差值太大 | 减小对应case的误差绝对值 |
| 误差方向反了 | case写反了 | 将所有误差值取反 |

**调整步骤：**
1. 进入模式4（调试模式），将小车放在黑线上
2. 手动移动小车，观察OLED显示的传感器状态（Hex）
3. 记录哪些状态的误差值需要调整
4. 在 `Huidu_Get_Error()` 中找到对应的 case，修改误差值
5. 重新编译测试

## 调参场景与策略

### 场景1：第一次调试（从零开始）

**步骤1：校准传感器（模式4）**

1. 按4次KEY1，进入模式4
2. 将小车放在黑线上，左右移动
3. 观察OLED显示：
   - 传感器二进制状态是否正确
   - 误差值方向是否正确（左负右正）
4. 如果方向反了，在 `Huidu_Get_Error()` 中将所有误差值取反

**步骤2：测试电机（模式2）**

1. 按2次KEY1，进入模式2
2. 观察小车是否直线前进
3. 检查OLED显示：
   - 左右轮速度是否接近（差异<10%）
   - 编码器计数是否正常增加
4. 如果速度差异大，调整电机PI参数（Kp, Ki）

**步骤3：测试转向（模式3）**

1. 按3次KEY1，进入模式3
2. 观察小车是否能左转
3. 确认差速控制正常

**步骤4：初步巡线（模式1）**

1. 将小车放在黑线中央
2. 按1次KEY1，进入模式1
3. 观察巡线效果：
   - 能否跟随黑线
   - 是否左右摆动
   - 弯道是否冲出去

**步骤5：调整巡线参数**

根据步骤4的现象，调整 `huidu.c` 中的参数：

```c
// 如果左右摆动严重
#define LINE_FOLLOW_KP  40.0f  // 减小Kp（50→40）
#define LINE_FOLLOW_KD  15.0f  // 增大Kd（10→15）

// 如果弯道冲出去
#define LINE_FOLLOW_KP  60.0f  // 增大Kp（50→60）
#define LINE_FOLLOW_BASE_SPEED  250.0f  // 减小速度（300→250）

// 如果转向慢
#define LINE_FOLLOW_KP  70.0f  // 增大Kp（50→70）
#define LINE_FOLLOW_KD  5.0f   // 减小Kd（10→5）
```

### 场景2：直线稳定，但弯道性能差

**现象：**
- 直线上能稳定跟随
- 到弯道就冲出去或反应慢

**原因分析：**
1. Kp 太小，转向力度不够
2. 速度太快，弯道来不及转
3. 误差值设置不合理

**调整策略：**

```c
// 方案1：增大Kp（提高转向力度）
#define LINE_FOLLOW_KP  70.0f  // 50→70

// 方案2：减小速度（留更多反应时间）
#define LINE_FOLLOW_BASE_SPEED  250.0f  // 300→250

// 方案3：增大弯道状态的误差值
// 在 Huidu_Get_Error() 中找到弯道状态（如单侧多个传感器触发）
case 0b00000111:  // L2 + L3 + L4（严重偏右）
    error = -3.0f;  // 原来是-2.5，增大到-3.0

case 0b11100000:  // R2 + R3 + R4（严重偏左）
    error = 3.0f;   // 原来是2.5，增大到3.0
```

### 场景3：弯道好，但直线摆动

**现象：**
- 弯道能顺利通过
- 直线上左右摆动（S型轨迹）

**原因分析：**
- Kp 太大，对小误差反应过度
- Kd 太小，没有抑制震荡

**调整策略：**

```c
// 方案1：减小Kp（降低转向敏感度）
#define LINE_FOLLOW_KP  30.0f  // 50→30

// 方案2：增大Kd（抑制震荡）
#define LINE_FOLLOW_KD  20.0f  // 10→20

// 方案3：减小中心附近状态的误差值
case 0b00001100:  // L1 + L2
    error = -0.5f;  // 原来是-1.0，减小到-0.5

case 0b00110000:  // R1 + R2
    error = 0.5f;   // 原来是1.0，减小到0.5
```

### 场景4：丢线后无法恢复

**现象：**
- 在弯道或断线处丢线
- 丢线后小车乱跑或停止

**原因分析：**
1. 丢线记忆逻辑错误
2. 极限误差值不够大
3. 速度太快，来不及找回线

**调整策略：**

```c
// 方案1：增大极限误差值
// 在 Huidu_Get_Error() 中找到丢线处理
case 0b00000000:  // 全白（完全丢线）
    if (last_valid_error > 0) {
        error = 6.0f;   // 原来是5.0，增大到6.0
    } else if (last_valid_error < 0) {
        error = -6.0f;  // 原来是-5.0，增大到-6.0
    }
    break;

// 方案2：减小速度（留更多寻线时间）
#define LINE_FOLLOW_BASE_SPEED  200.0f  // 300→200
```

## 常见问题与解决方案

### Q1：小车一直原地打转

**原因：** 误差方向反了

**解决：**
1. 进入模式4，观察误差值方向
2. 如果左移时误差为正（应该是负），说明方向反了
3. 在 `Huidu_Get_Error()` 中将所有 `error = X` 改为 `error = -X`

---

### Q2：左右轮速度差异大

**原因：** 左右电机PI参数不一致或机械差异

**解决：**
```c
// 在 motor.c 中分别调整左右轮参数
Motor_PI_TypeDef Motor_Left = {
    .kp = 8.0f,   // 左轮参数
    .ki = 0.5f,
};

Motor_PI_TypeDef Motor_Right = {
    .kp = 9.0f,   // 右轮参数（可以和左轮不同）
    .ki = 0.6f,
};
```

---

### Q3：编码器计数为0

**原因：** 编码器中断未配置或硬件连接问题

**解决：**
1. 检查 `ti_msp_dl_config.h` 中是否有 `HUI_DU_LX_IIDX` 和 `DC_MOTOR_XXX_IIDX` 定义
2. 检查编码器供电和信号线连接
3. 用示波器测量编码器输出

---

### Q4：参数改了不生效

**原因：** 忘记重新编译

**解决：**
1. 修改参数后，点击 CCS 的 "Build" 按钮
2. 等待编译完成
3. 下载到开发板
4. 复位重启
