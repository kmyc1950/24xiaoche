# 右轮编码器和OLED显示修复报告

## 问题描述

程序运行后：
- OLED 显示屏显示左轮速度 -325 mm/s
- 右轮速度显示为 0 mm/s
- 但右轮电机实际在正常运行

## 问题诊断

### 问题1：右轮编码器中断未配置 ❌

**根本原因：**
在 `Debug/ti_msp_dl_config.h` 中：
- 左轮编码器 AA 引脚有中断配置：`DC_MOTOR_LEFT_AA_IIDX` 已定义
- 右轮编码器 BA 引脚**没有中断配置**：`DC_MOTOR_RIGHT_BA_IIDX` 不存在

**硬件配置对比：**
```c
// 左轮编码器（GPIOA.9）- 有中断
#define DC_MOTOR_LEFT_AA_PORT    (GPIOA)
#define DC_MOTOR_LEFT_AA_IIDX    (DL_GPIO_IIDX_DIO9)  // ✅ 中断已配置
#define DC_MOTOR_LEFT_AA_PIN     (DL_GPIO_PIN_9)

// 右轮编码器（GPIOB.7）- 无中断
#define DC_MOTOR_RIGHT_BA_PIN    (DL_GPIO_PIN_7)
// ❌ 缺少：DC_MOTOR_RIGHT_BA_IIDX 定义
```

**导致后果：**
- `GROUP1_IRQHandler` 只处理左轮编码器中断
- `encoder_counter_right` 永远是 0
- `Motor_Right.current_speed` 始终计算为 0

### 问题2：main.c 中没有显示代码 ❌

**原因：**
- main.c 中所有 OLED 显示代码都被注释掉了
- 无法看到实时速度和编码器计数

### 问题3：左轮速度为负数 -325

**可能原因：**
1. 编码器 A/B 相接线反了
2. 电机实际反转但方向控制认为是正转
3. 编码器方向判断逻辑与实际相位相反

## 修复方案

### 修复1：右轮编码器轮询读取（临时方案）

由于 SysConfig 中未配置右轮编码器中断，采用**软件轮询方式**读取：

**motor.c:149 - calculate_speed() 修改：**

```c
void calculate_speed(uint8_t motor_id)
{
    if (motor_id == 1) {
        // 左轮速度计算（中断方式）
        Motor_Left.current_speed = (float)encoder_counter_left * 
            WHEEL_CIRCUMFERENCE_MM / ENCODER_PPR / PID_SAMPLE_TIME_S;
        encoder_counter_left = 0;

    } else if (motor_id == 2) {
        // 右轮速度计算（轮询方式 - 临时方案）
        // TODO: 需要在 SysConfig 中配置 DC_MOTOR_RIGHT_BA 为 GPIO 中断

        // 轮询读取右轮编码器BA引脚状态
        static uint8_t last_state_right_ba = 0;
        uint32_t current_state_ba = DL_GPIO_readPins(DC_MOTOR_RIGHT_PORT, 
                                                      DC_MOTOR_RIGHT_BA_PIN);

        // 检测上升沿
        if (current_state_ba != 0 && last_state_right_ba == 0) {
            // BA 上升沿，读取 BB 相判断方向
            uint32_t pin_bb = DL_GPIO_readPins(DC_MOTOR_RIGHT_PORT, 
                                                DC_MOTOR_RIGHT_BB_PIN);

            if (pin_bb == 0) {
                encoder_counter_right++;  // 正转
            } else {
                encoder_counter_right--;  // 反转
            }
        }
        last_state_right_ba = (current_state_ba != 0) ? 1 : 0;

        // 计算速度
        Motor_Right.current_speed = (float)encoder_counter_right * 
            WHEEL_CIRCUMFERENCE_MM / ENCODER_PPR / PID_SAMPLE_TIME_S;
        encoder_counter_right = 0;
    }
}
```

**轮询方式的局限性：**
- ⚠️ 每 50ms 只采样一次，可能漏掉部分脉冲（中断方式不会漏）
- ⚠️ 高速运行时测速精度会下降
- ⚠️ 如果 50ms 内脉冲超过采样窗口，会丢失计数

**为什么暂时能用？**
- 50ms 采样周期内，编码器脉冲数通常不会太多
- 对于低中速运行（300-500 mm/s），轮询方式基本可用
- 用于快速验证功能，确认右轮电机和 PI 控制是否正常

## 长期解决方案（推荐）

### 在 SysConfig 中配置右轮编码器中断

**操作步骤：**

1. **打开 SysConfig 文件**
   - 双击项目中的 `.syscfg` 文件

2. **配置 GPIOB.7（DC_MOTOR_RIGHT_BA）**
   - 找到 `DC_MOTOR_RIGHT` 组 → `BA` 引脚
   - Mode: `Input with internal pull-up/pull-down`
   - Internal Resistor: `Pull Up`
   - Interrupt: `Enable`
   - Interrupt Trigger: `Rising Edge`（上升沿触发）
   - Interrupt Priority: 与 GPIOA 相同优先级

3. **重新生成代码**
   - 保存 `.syscfg` 文件
   - SysConfig 会自动生成 `DC_MOTOR_RIGHT_BA_IIDX` 定义
   - 查看 `ti_msp_dl_config.h` 确认生成成功

4. **修改 motor.c 中的中断处理**

需要增加 GPIOB 中断处理函数（假设右轮在 GROUP2）：

```c
// 需要查看 ti_msp_dl_config.h 确认右轮编码器属于哪个中断组
void GROUP2_IRQHandler(void)  // 或其他 GROUP 名称
{
    uint32_t gpio_iidx = DL_GPIO_getPendingInterrupt(DC_MOTOR_RIGHT_PORT);
    
    switch (gpio_iidx)
    {
    case DC_MOTOR_RIGHT_BA_IIDX:
        // 右轮编码器BA相中断（上升沿）
        {
            uint32_t pin_bb = DL_GPIO_readPins(DC_MOTOR_RIGHT_PORT, 
                                                DC_MOTOR_RIGHT_BB_PIN);
            if (pin_bb == 0) {
                encoder_counter_right++;  // 正转
            } else {
                encoder_counter_right--;  // 反转
            }
        }
        break;
        
    default:
        break;
    }
}
```

5. **在 main.c 中使能 GPIOB 中断**

```c
// 查找右轮编码器对应的中断号（在 ti_msp_dl_config.h 中）
NVIC_EnableIRQ(GPIO_MULTIPLE_GPIOB_INT_IRQN);  // 示例，具体名称需确认
```

6. **移除轮询代码**

将 `calculate_speed(2)` 中的轮询代码改回简单的速度计算：

## 关键调试技巧

### 1. 监控编码器计数

OLED 第3、4行显示的 `LC` 和 `RC` 是实时编码器计数：
- 正常运行时应该看到数字不断变化
- 如果某个计数始终为 0，说明对应编码器没有工作

### 2. 检查硬件连接

**编码器引脚定义：**
```
左轮编码器：
- AA (A相): GPIOA.9
- AB (B相): GPIOA.8

右轮编码器：
- BA (A相): GPIOB.7
- BB (B相): GPIOB.6
```

### 3. 示波器验证

用示波器测量编码器输出：
- A 相应该有连续的方波信号
- B 相与 A 相有 90° 相位差
- 电机转动时波形频率应该随速度变化
