# 按键和中断宏定义修复说明

## 问题描述

在重构过程中发现了以下问题：

### 1. 中断处理函数重复定义
- **问题**：`GROUP1_IRQHandler` 在 `key.c` 和 `motor.c` 中都有定义
- **原因**：按键（KEY1）和左电机编码器（AA）都连接到 GPIOA，共享同一个 GPIO 中断
- **后果**：会导致编译链接错误（函数重复定义）

### 2. 中断宏定义缺失
- **问题**：main.c 中使用了 `KEY_INT_IRQN` 宏，但 ti_msp_dl_config.h 中未定义
- **原因**：SysConfig 生成的配置只定义了 `GPIO_MULTIPLE_GPIOA_INT_IRQN`
- **后果**：会导致编译错误（未定义的标识符）

### 3. 重复使能中断
- **问题**：main.c 中分别调用了 `NVIC_EnableIRQ(KEY_INT_IRQN)` 和 `NVIC_EnableIRQ(DC_MOTOR_INT_IRQN)`
- **原因**：这两个中断实际上是同一个 GPIOA 中断
- **后果**：虽然不会报错，但逻辑上冗余

---

## 修复方案

### ✅ 修复 1：合并中断处理函数

**修改文件：key.c**
- **删除**：原有的 `GROUP1_IRQHandler()` 函数定义
- **添加**：说明注释，指出中断处理已移到 motor.c

**修改文件：motor.c**
- **保留**：`GROUP1_IRQHandler()` 函数
- **增强**：在同一个中断处理函数中同时处理：
  - `DC_MOTOR_LEFT_AA_IIDX`：左轮编码器A相中断
  - `KEY_KEY1_IIDX`：按键中断（保留原有逻辑）

**修改后的 motor.c 中断处理函数：**

```c
void GROUP1_IRQHandler(void)
{
    uint32_t gpio_iidx = DL_GPIO_getPendingInterrupt(DC_MOTOR_LEFT_AA_PORT);

    switch (gpio_iidx)
    {
    case DC_MOTOR_LEFT_AA_IIDX:
        // 左轮编码器A相中断
        {
            uint32_t pin_b = DL_GPIO_readPins(DC_MOTOR_LEFT_AB_PORT, DC_MOTOR_LEFT_AB_PIN);
            if (pin_b == 0) {
                encoder_counter_left++;  // 正转
            } else {
                encoder_counter_left--;  // 反转
            }
        }
        break;

    case KEY_KEY1_IIDX:
        // 按键中断（保留原有逻辑）
        {
            extern int status;
            status = (status + 1) % 3;
        }
        break;

    default:
        break;
    }
}
```

### ✅ 修复 2：添加按键中断宏定义

**修改文件：key.h**
- **添加**：`KEY_INT_IRQN` 宏定义，映射到 `GPIO_MULTIPLE_GPIOA_INT_IRQN`

```c
// 按键中断宏定义（与编码器共享GPIOA中断）
#define KEY_INT_IRQN            GPIO_MULTIPLE_GPIOA_INT_IRQN
```

### ✅ 修复 3：简化中断使能代码

**修改文件：main.c**
- **删除**：重复的 `NVIC_EnableIRQ(KEY_INT_IRQN)` 调用
- **保留**：只调用一次 `NVIC_EnableIRQ(DC_MOTOR_INT_IRQN)`
- **添加**：说明注释

**修改后的 main.c：**

```c
// 使能GPIOA中断（按键KEY1和左电机编码器AA共享此中断）
NVIC_EnableIRQ(DC_MOTOR_INT_IRQN);  // 等价于 GPIOA_INT_IRQn
```

## 中断源共享说明

### GPIOA 中断共享的外设

根据 ti_msp_dl_config.h 的配置：

| 引脚 | 功能 | 中断索引 | 用途 |
|-----|------|---------|------|
| GPIOA.2 | KEY1 | `KEY_KEY1_IIDX` (DL_GPIO_IIDX_DIO2) | 按键输入 |
| GPIOA.9 | AA | `DC_MOTOR_LEFT_AA_IIDX` (DL_GPIO_IIDX_DIO9) | 左轮编码器A相 |

这两个引脚都在 **GPIOA 端口**上，因此它们共享同一个 GPIO 中断向量 `GPIOA_INT_IRQn`。

### 中断处理流程

```
硬件触发 → GPIOA_INT_IRQn → GROUP1_IRQHandler() → 读取中断索引
                                                         ↓
                               +-------------------------+-------------------------+
                               |                                                   |
                               v                                                   v
                    KEY_KEY1_IIDX (按键中断)                      DC_MOTOR_LEFT_AA_IIDX (编码器中断)
                               ↓                                                   ↓
                    status = (status + 1) % 3                      encoder_counter_left++/-
```

## 注意事项

### 如果需要添加右轮编码器中断

右轮编码器（BA/BB）在 **GPIOB** 上，需要：

1. 在 SysConfig 中配置 `DC_MOTOR_RIGHT_BA_PIN` (GPIOB.7) 为 GPIO 输入中断
2. 在代码中添加 GPIOB 中断处理函数（**不是** GROUP1_IRQHandler）

**示例代码框架：**

```c
// 需要在 motor.c 中添加
void GPIOB_IRQHandler(void)  // 注意：GPIOB 的中断处理函数名不同
{
    uint32_t gpio_iidx = DL_GPIO_getPendingInterrupt(DC_MOTOR_RIGHT_BA_PORT);

    switch (gpio_iidx)
    {
    case DC_MOTOR_RIGHT_BA_IIDX:  // 需要在 SysConfig 中定义
        {
            uint32_t pin_b = DL_GPIO_readPins(DC_MOTOR_RIGHT_BB_PORT, DC_MOTOR_RIGHT_BB_PIN);
            if (pin_b == 0) {
                encoder_counter_right++;
            } else {
                encoder_counter_right--;
            }
        }
        break;

    default:
        break;
    }
}
```

并在 main.c 中使能：
```c
NVIC_EnableIRQ(GPIOB_INT_IRQn);  // 使能 GPIOB 中断
```
