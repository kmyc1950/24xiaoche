# 巡线车代码重构说明 - 穷举法替代加权平均

**重构日期：** 2026-07-03  
**重构原因：** 加权平均法在多传感器触发时误差错误叠加，影响巡线精度

---

### 核心改动

1. ✅ **废弃加权平均算法** - 改为穷举法（switch-case）

## 🔄 算法对比

### 旧算法：加权平均法

**原理：**
```
误差 = Σ(传感器状态 × 权重) / Σ(传感器状态)
```

**问题：**
- 多个传感器同时触发时，误差值是加权平均
- 例如：传感器 L1+L2+L3 同时触发，误差 = (-0.5-1.5-2.5)/3 = -1.5
- 但实际情况可能是宽线或斜线，-1.5 不一定准确
- 无法精确控制特定传感器组合的行为

### 新算法：穷举法

**原理：**
```c
switch (sensor_data) {
    case 0b00011000:  // L1 + R1（中间两个）
        error = 0.0f;
        break;
    case 0b00001100:  // L1 + L2（偏右）
        error = -1.0f;
        break;
    // ... 穷举所有合理状态
}
```

**优势：**
- 每种传感器组合都有明确的误差值
- 可以根据实际测试调整每个状态的误差
- 对宽线、交叉路口等特殊情况可以特殊处理
- 逻辑清晰，易于调试

## 🎯 丢线记忆功能

### 原理

当检测到全白（0x00）时：
1. 读取上一次有效的误差值 `last_valid_error`
2. 根据正负输出极限误差：
   - `last_valid_error > 0` → 输出 `+5.0`（大角度右转寻线）
   - `last_valid_error < 0` → 输出 `-5.0`（大角度左转寻线）
   - `last_valid_error == 0` → 输出 `0.0`（保持直行）

### 代码实现

```c
case 0b00000000:  // 全白（完全丢线）
    {
        if (last_valid_error > 0) {
            // 上次黑线在右侧（小车偏左），继续大角度右转寻线
            error = 5.0f;
        } else if (last_valid_error < 0) {
            // 上次黑线在左侧（小车偏右），继续大角度左转寻线
            error = -5.0f;
        } else {
            // 上次误差为0（不太可能），默认保持直行
            error = 0.0f;
        }
    }
    break;
```

### 效果

- 丢线后小车会保持原有转向趋势，大角度回转寻线
- 避免丢线后小车乱跑或停止
- 提高弯道和断线的通过性

## 🚗 自动巡线控制函数

### 函数签名

```c
void Huidu_LineFollow_Task(void);
```

### 控制流程

```
1. 读取灰度传感器
   ↓
2. 计算位置误差（穷举法）
   ↓
3. PD控制算法
   control_output = Kp × error + Kd × (error - last_error)
   ↓
4. 差速控制
   left_speed  = BASE_SPEED - control_output
   right_speed = BASE_SPEED + control_output
   ↓
5. 速度限幅
   ↓
6. 更新电机目标速度
   Motor_Left.target_speed = left_speed
   Motor_Right.target_speed = right_speed
```

### PD控制参数

```c
#define LINE_FOLLOW_KP          50.0f   // 比例系数
#define LINE_FOLLOW_KD          10.0f   // 微分系数
#define LINE_FOLLOW_BASE_SPEED  300.0f  // 基础速度（mm/s）
```

**参数调试建议：**

| 参数 | 作用 | 过大 | 过小 |
|-----|------|------|------|
| Kp | 转向力度 | 震荡、左右摆动 | 响应慢、转弯不及时 |
| Kd | 稳定性 | 反应迟钝、过度阻尼 | 震荡、欠阻尼 |
| BASE_SPEED | 前进速度 | 冲出弯道 | 速度太慢 |

**调试顺序：**
1. 先将 Kd 设为 0，只调 Kp，找到能跟线但有轻微震荡的 Kp 值
2. 再逐渐增加 Kd，抑制震荡，直到稳定跟线
3. 最后微调基础速度，在保证稳定的前提下提高速度

## 📱 主函数多模式控制

### 模式定义

```c
int run_mode = 0;  // 运行模式

// 模式说明：
// 0 = 停止
// 1 = 自动巡线
// 2 = 直行测试
// 3 = 转弯测试
// 4 = 调试模式（显示传感器原始数据）
```

### 模式切换

- 通过按键 KEY1 切换模式：`run_mode = (run_mode + 1) % 5`
- 按键中断在 `motor.c` 的 `GROUP1_IRQHandler` 中处理

### 各模式功能

**模式0：停止**
- 停止所有电机
- OLED 显示 "Mode: STOP"

**模式1：自动巡线** ⭐
- 调用 `Huidu_LineFollow_Task()` 实现自动巡线
- OLED 实时显示：
  - 传感器状态（十六进制）
  - 位置误差
  - 左右轮目标速度

**模式2：直行测试**
- 固定速度 300mm/s 直行
- 用于测试电机和编码器

**模式3：转弯测试**
- 左轮 200mm/s，右轮 400mm/s
- 用于测试差速转向

**模式4：调试模式**
- 只显示传感器数据，电机停止
- 用于校准传感器和误差值

### 步骤1：校准传感器（模式4）

1. 进入模式4（调试模式）
2. 将小车放在黑线上，左右晃动
3. 观察 OLED 显示：
   - 传感器二进制状态是否正确
   - 误差值方向是否正确（左负右正）

### 步骤2：测试电机（模式2）

1. 进入模式2（直行测试）
2. 观察小车是否能直线前进
3. 检查 OLED 显示：
   - 左右轮速度是否接近
   - 编码器计数是否正常增加
4. 如果速度相差太大，检查编码器方向或电机接线

### 步骤3：测试转向（模式3）

1. 进入模式3（转弯测试）
2. 观察小车是否能左转（左慢右快）
3. 确认差速控制正常工作

### 步骤4：巡线测试（模式1）

1. 将小车放在黑线上
2. 进入模式1（自动巡线）
3. 观察：
   - 小车是否能自动跟随黑线
   - 误差值和速度调整是否合理
   - 弯道是否能顺利通过
   - 丢线后是否能自动寻线

### 步骤5：参数调优

如果巡线效果不好，调整 `huidu.c` 中的参数：

**震荡太大（左右摆动）：**
- 减小 `LINE_FOLLOW_KP`（如 50 → 40）
- 增大 `LINE_FOLLOW_KD`（如 10 → 15）

**响应太慢（转弯不及时）：**
- 增大 `LINE_FOLLOW_KP`（如 50 → 60）
- 减小 `LINE_FOLLOW_KD`（如 10 → 5）

**弯道冲出去：**
- 降低 `LINE_FOLLOW_BASE_SPEED`（如 300 → 250）
- 增大 `LINE_FOLLOW_KP`

**丢线后找不回来：**
- 检查 `case 0b00000000` 中的极限误差方向
- 增大极限误差值（如 ±5.0 → ±6.0）

## ⚠️ 常见问题

### Q1：小车一直原地打转

**原因：** 误差方向反了

**解决：**
1. 进入模式4，将小车放在线左侧，观察误差值
2. 如果误差是正数（预期应该是负数），说明方向反了
3. 修改 `Huidu_Get_Error()` 中所有 case 的误差值，全部取反

### Q2：丢线后无法恢复

**原因：** 丢线记忆逻辑错误或极限误差不够大

**解决：**
1. 检查 `case 0b00000000` 中的极限误差方向
2. 尝试增大极限误差值（±5.0 → ±6.0）
3. 检查 `last_valid_error` 是否正确更新

### Q3：某些传感器组合没有穷举到

**症状：** 进入 `default` 分支，使用上一次误差

**解决：**
1. 在 OLED 上显示传感器状态，找出未穷举的组合
2. 在 `Huidu_Get_Error()` 中添加对应的 case
3. 根据实际情况赋予合理的误差值

### Q4：直线跑不稳，左右摆动

**原因：** Kp 太大或 Kd 太小

**解决：**
1. 减小 `LINE_FOLLOW_KP`（如 50 → 40 → 30）
2. 增大 `LINE_FOLLOW_KD`（如 10 → 15 → 20）
3. 逐步调整，观察效果
